🔥 Redis 高速缓存

高性能 key-value 内存数据库，亚毫秒级响应，常用于实时计算结果缓存、计数器、排行榜与会话管理。

核心特点

内存操作、数据类型丰富、支持持久化与分布式集群

⚡ 高性能

数据驻留内存，读写速度极快，轻松达到十万级 QPS。

📦 数据类型丰富

支持 String、Hash、List、Set、Sorted Set 等多种数据结构。

🕔 TTL 过期

通过 EXPIRE 设置 key 生存时间，支持验证码、会话等临时数据。

💾 持久化

RDB 快照与 AOF 日志两种机制，平衡恢复速度与数据安全。

💨 Pipeline

批量发送命令，减少网络往返，大幅提升吞吐。

🎯 实时落地

Flink/Spark Streaming 计算结果写入 Redis，供前端实时查询。

数据类型与常用命令

不同数据结构适配不同业务场景

String（字符串）

最基本数据类型，适合缓存、计数器、分布式锁等场景。

命令功能示例
SET设置 key-valueSET name "zhang"
GET获取 valueGET name
INCR原子自增 1INCR count
INCRBY按指定值自增INCRBY count 5
MSET/MGET批量设置/获取MSET k1 v1 k2 v2

Hash（哈希）

字段映射表，适合存储对象，如用户信息。

命令功能示例
HSET设置字段值HSET user:1 name "zhang"
HGET获取字段值HGET user:1 name
HGETALL获取所有字段HGETALL user:1
HINCRBY字段自增HINCRBY user:1 age 1
HDEL删除字段HDEL user:1 age

List（有序双向链表）

适合最新 N 个数据、消息队列等场景。

命令功能示例
LPUSH左侧插入LPUSH mylist "a"
RPUSH右侧插入RPUSH mylist "b"
LPOP/RPOP左右弹出LPOP mylist
LRANGE获取范围元素LRANGE mylist 0 -1
LLEN获取列表长度LLEN mylist

Set（不重复无序集合）

适合标签、共同好友、去重等场景。

命令功能示例
SADD添加元素SADD myset "a"
SMEMBERS获取所有元素SMEMBERS myset
SISMEMBER判断是否存在SISMEMBER myset "a"
SINTER交集SINTER set1 set2
SDIFF差集SDIFF set1 set2

Sorted Set（有序集合）

每个元素关联 score，自动排序，适合排行榜、热搜。

命令功能示例
ZADD添加元素与分数ZADD myzset 100 "m1"
ZRANGE按分数升序ZRANGE myzset 0 -1
ZREVRANGE按分数降序ZREVRANGE myzset 0 9
ZINCRBY分数自增ZINCRBY myzset 10 "m1"
ZREM删除元素ZREM myzset "m1"

持久化机制

RDB 快照与 AOF 日志的对比与选择

RDB（快照）

  • 触发命令：SAVE / BGSAVE
  • 机制：定时快照保存内存数据
  • 文件：二进制，紧凑
  • 恢复速度：快
  • 数据安全：可能丢失最后一次快照后的数据

AOF（日志）

  • 开启：appendonly yes
  • 机制：记录每个写操作
  • 文件：文本日志，较大
  • 恢复速度：相对慢
  • 数据安全：默认 everysec，最多丢 1 秒数据
